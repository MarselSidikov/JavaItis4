int - 4 байта
char - 2 байта
double - 8 байл
boolean - 1 байт

Приведение типов
Неявное приведение:
double x = 7;
7 - int, но он меньше, чем double, поэтому данные потеряны не будут, и поэтому преобразование неявное.
int c = 'A';
Явное преобразование:
int x = (int)7.4;

Массивы - набор переменных, доступных по индексу.
int a[] = new int[5];
Массивы - ссылочный тип, то есть значение переменной 'a' указывает на первый элемент массива.

Класс - что-то общее, объект - экземпляр класса.


Наследование - механизм ООП, позволяющий объектам класса потомка
копировать и переопределять состояние и поведение объектов класса предка.

Восходящее преобразование - механизм (элемент наследования).
Позволяет преобразовать тип объектной переменной класса потомка к типу класса предка.

Пусть есть класс A с методом x(). Если класс B тоже определяет метод x(), и при этом
является наследником класса A, то говорят, что метод x() - переопределен.
То есть поведение класса A сохраняется в классе B, но при этом реализовано оно по другому.

Абстрактный класс - класс, объекты которого создавать нельзя
Модификатор abstract - если вешается на класс,
то это означает, что класс является абстрактным.
Если вешается на метод - то значит, что у метода нет реализации.

Интерфейс - (для школьников) - класс, в котором нет полей, и все методы
абстрактные.
Интерфейс - (для нормальных людей) - инкапсуляция поведения. Таким образом,
интерфейс говорит, что все классы, который данный интерфейс
наследуют (НЕ ПРАВИЛЬНО ТАК ГОВОРИТЬ) имплементируют, будут обязательно
содержать методы, заявленные в интерфейсе.

В JAVA класс не может наследоваться более, чем от одного класса.
А класс может имплементировать более одного интерфейса.

Класс Object - божественный класс.
Предок всех классов.
Методы с разными формальными параметрами и одинаковыми названиями называются перегруженными.
public String toString() - возвращает строковое представление объекта
public String toString() - возвращает строковое представление объекта
По умолчанию: имякласса@хешкод
== - сравниванивает по значению
public boolean equals(Object obj) - метод сравнения.
В классе Object просто сравнивает ссылки через ==.
Для корректного сравнения необходимо переопределить.

StringPool - пул строк, хранит литеральные строки, что повышает производительность.
То есть, если в коде встречаются одинаковые литеральные строки
то для них создается один объект и помещается в StringPool

Полиморфизм - принцип ООП, позволяющий работать с объектами
разных типов таким образом, будто они принадлежат одному типу.
